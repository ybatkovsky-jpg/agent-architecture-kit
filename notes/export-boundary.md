# Export boundary

This repository should contain sanitized architectural patterns, not raw private operations.

## Keep out by default
- live credentials
- private chat identifiers
- personal long-term memory files
- environment-specific secrets
- raw operational logs unless deliberately redacted
- user-specific notes that do not generalize

## Prefer to export
- public-safe architecture principles
- generalized patterns
- redacted examples
- generic schemas
- reusable evaluation approaches
- implementation-neutral docs where possible
