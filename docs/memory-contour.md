# Memory Contour

## What memory is

Memory is a selective continuity layer that improves future execution quality.

It is not:
- a full transcript archive;
- a dump of every conversation;
- a replacement for task state;
- a replacement for deep artifacts.

## What should be distilled

- decisions
- patterns
- anti-patterns
- blockers with reuse value
- durable references
- stable operating rules

## What should usually stay out

- raw chatter
- every tiny step
- transient noise
- duplicate summaries

## Distillation triggers

Typical triggers:
- a decision was accepted;
- a bounded slice produced a reusable lesson;
- a blocker exposed a systemic boundary;
- a canonical artifact was created;
- a workstream was closed and context would otherwise be lost.

## Truth hierarchy

Memory does not automatically win every conflict.

Examples:
- for task status, task state should usually win;
- for user preferences, curated memory may win;
- for deep implementation detail, linked artifacts may win.

## Goal

The goal of memory is not to remember everything.
The goal is to remember the right things in a form that survives future sessions and improves later decisions.
