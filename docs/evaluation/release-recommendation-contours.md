# Release Recommendation Contours

## Main rule

Release readiness is not the same as architectural promise.

A contour may be:
- architecturally strong;
- classifier-improved;
- partially repaired in bounded families;
- and still not ready for broad release.

## Example release stance

A correct release recommendation can remain `continue hardening` even after meaningful improvements if high-risk lanes are not yet cleanly closed.

Typical blockers:
- continuation can still resume from the wrong basis;
- audit can still point to the wrong proof artifact;
- preference recall is improved but not yet safely canonicalized;
- status queries are better but not yet strongly re-proven.

## Key distinction

Improved is not the same as rollout-ready.
